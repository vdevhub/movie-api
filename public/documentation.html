Movie API Documentation

API Intro

Movie API is a REST API for an application that interacts with a database that stores data about different movies.

Supported Requests

Business Logic URL Endpoints HTTP Method Request Body Format Response Body Format
Return a list of ALL movies to the user /movies GET None A JSON object holding data about all the movies: 
            [
              {
                "title": "Pearl Harbor",
                "year": "2001",
                "director": {
                    "name": "Michael Bay",
                    "bio": "It is a director."
                },
                "genre": {
                    "name": "Drama",
                    "description": "Some genre"
                }
              },
              {
                "title": "Suite Française",
                "year": "2014",
                "director": {
                    "name": "Saul Dibb",
                    "bio": "It is a director."
                },
                "genre": {
                    "name": "Drama",
                    "description": "Some genre"
                }
              }
            ]
Return data about a single movie by title to the user /movies/[movieTitle] GET None A JSON object holding data about the requested movie: 
              {
                "title": "Eat, Pray, Love",
                "year": "2010",
                "director": {
                    "name": "Ryan Murphy",
                    "bio": "It is a director."
                },
                "genre": {
                    "name": "Drama",
                    "description": "Some genre"
                }
              }
Return data about a genre (description) by name (e.g., “Thriller”) /movies/genres/[genreName] GET None A JSON object holding data about the requested genre: 
              {
                "name": "Drama",
                "description": "Some genre"
              }
Return data about a director (bio, birth year, death year) by name /movies/director/[directorName] GET None A JSON object holding data about the requested director: 
              {
                "name": "Michael Bay",
                "bio": "It is a director."
              }
Register a new user /users POST A JSON object holding data about the user to be added, for example: 
            {
              "name": "Mary",
              "favouriteMovies": ["Titanic", "Ocean's Eleven"]
            }
A JSON object holding data about the newly added user and their new ID: 
              {
                "name": "Mary",
                "favouriteMovies": [],
                "id": "99f22720-3b61-4294-bada-fe2fff4dfb82"
              }
Update user information /users/[id] PUT A JSON object holding data to be updated for the user, for example: 
            {
              "name": "Paul White",
              "email": "paul.white@gmail.com"
            }
A JSON object holding data about the updated user: 
            {
              "id": "1",
              "name": "Paul White",
              "email": "paul.white@gmail.com",
              "favoriteMovies": [
                   "War"
               ]
            }
Add a movie to the user's list of favorites /users/[id]/[movieTitle] POST None A text message indicating whether the movie was successfully added to the user's favorite movies list
Remove a movie from the user's list of favorites /users/[id]/[movieTitle] DELETE None A text message indicating whether the movie was successfully removed from the user's favorite movies list
Deregister a user /users/[id] DELETE None A text message indicating whether the user with the specified ID was removed
